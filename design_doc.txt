StepGuard: Iterative Process Monitoring and Targeted Suffix Repair for Improving LLM Reasoning

Stepguard is a research pipeline for improving LLM Reasoning through the monitoring of step-by-step reasoning traces to identify the earliest risky step (first reasoning step whose risk score crosses the risk threshold) and applying targeted suffix repair. 

Proposal:
LLMs often produce reasoning steps that look reasonable but contain:
unsupported assumptions
invalid logical jumps
contradictions
logical gaps
Rather than evaluating only the final answer, StepGuard takes things a step further by evaluating the reasoning process step-by-step.
Core pipeline:
Generate reasoning trace + final answer 
Split trace into reasoning steps
Score each step (verifier + contradiction; evidence support later)
Compute per-step risk and find earliest risky step
Repair by regenerating the suffix from the earliest risky step
Rescore the repaired trace
Iterate repairs until improvement plateaus, risk worsens, or max_iters reached
Evaluate final answer correctness and process risk reduction

Data Format (JSON):
Original trace schema:
{
 "id": "...", //question id
 "question": "...", //source question
 "task": "math|nonmath", //what kind of task
 "source": "...", //dataset
 "gold_answer": "...", //ideal answer according to dataset
 "evidence": [...], //currently optional
 "model": "...", //model name (gpt-5.4-mini, gemini-3.1-flash, etc)
 "prompt_id": "...", //unique ID for current prompt
 "model_trace": "...", // step-by-step reasoning output
 "model_answer": "..." //final answer given by trace
}

Scored schema:

{
 "steps": ["Step 1: ...", "Step 2: ...", ...], //all steps but answer"
 "scores": {
   "verifier": [0.0, 0.2, ...], //p_wrong per step
   "contradiction": [0.0, 0.1, ...], //P(contradiction) per step
   "evidence_support": null //reserved for non-math later
 },
 "risks": [0.1, 0.3, ...], // combined per-step risk
 "earliest_bad_step": null | 0 | 1 | … // first step that exceeds risk threshold
}

Repaired schema:
{
"repaired_trace": "...", //full repaired trace (prefix + regenerated suffix)
"repaired_answer": "...", //final answer from repaired trace
"logs": {
"repair": {
"performed": true|false,
"k": 1,
"next_step_number": 2
}
}
}

Iterative repair schema
{
  "final_trace": "...",
  "final_answer": "...",
  "final_steps": [...],
  "final_scores": {...},
  "final_risks": [...],
  "final_earliest_bad_step": 1,
  "logs": {
    "iterative_repair": {
      "max_iters": 2,
      "improvement_threshold": 0.02,
      "risk_threshold": 0.1,
      "stop_reason": "risk_worsened",
      "iterations": [...]
    }
  }
}


CLI Commands:

Prepare-dataset:
Exports dataset into JSON format for pipeline use

Generate-traces:
Reads JSON dataset, and generates reasoning trace + answer for every question using chosen provider, returns output in generated.jsonl

Score-traces:
Reads generated.jsonl and calculates verifier and contradiction scores for each step in the trace. It then uses a combination of these scores to assign a risk value. If step risk value is above threshold, it is designated a bad step. The first of these bad steps is the chosen step for earliest bad step for suffix generation, returns scored.jsonl

Repair-traces:
Reads scored.jsonl and finds first bad step, it then keeps all the steps before it and regenerates all steps after including itself, outputs repaired.jsonl. Repairs only once.

Iterative-repair:
Reads generated.jsonl and runs a full repair loop. Runs score-traces to find the earliest risky step, then runs Repair-traces to repair traces. Compares old risk to new risk, and accepts repair if risk does not worsen. Repeats if improvement meets improvement_threshold while max_iters has not been reached, outputs iterative_repaired.jsonl.

Evaluate:
Reads iterative_repaired.jsonl and computes correctness, risk, and repair metrics, outputs metrics.json and evaluation_records.jsonl

Run-pipeline:
Runs stages in order:
generate → iterative repair → evaluate

Scoring:

Contradiction score:
For each step i, up to 2 previous steps are defined as premise.
RoBERTa-NLI then returns probabilities for contradiction.
Scoring range: 0 -> no contradiction detected, 1 -> strong contradiction detected 

Verifier score:
For each trace, call the judge once with the question and the list of steps.
Scoring range: 0 ->step judged valid, 1 -> step judged invalid

Risk Score:
Combine contradiction and verifier scores, clipped to between 0 and 1.
Experiments will modify this.

Pipeline logic:
-Contradiction Scoring:
For each step i in steps:
    If i == 0:
        contradiction_score = 0.0
    Else:
        premise = previous 1-2 steps
        hypothesis = current step

        contradiction_score = P(contradiction | premise, hypothesis)
        using RoBERTa-MNLI

    store contradiction_score for step i

-Verifier Scoring:
Given question and steps:
    Send question + full step list to LLM judge

    Judge returns:
        step_index
        verdict
        p_wrong

    For each step i:
        verifier_score[i] = p_wrong[i]

    If judge fails:
        verifier_score[i] = 0.5

-Risk Scoring:
For each step i:
    verifier = verifier_score[i]
    contradiction = contradiction_score[i]

    risk[i] = min(1.0, verifier + contradiction)

    If risk[i] > risk_threshold:
        mark step as risky

-Earliest Risky Step Detection:
earliest_bad_step = null

For each step i in order:
    If risk[i] > risk_threshold:
        earliest_bad_step = i
        stop searching

If no step crosses threshold:
    earliest_bad_step remains null

-Iterative Repair Loop:
current_trace = original model_trace
current_answer = original model_answer

Score current_trace
Record original risk summary

For iter from 1 to max_iters:
    If there is no earliest risky step:
        stop with reason "no_bad_step"

    k = earliest risky step
    prefix = steps before k

    Generate candidate suffix from step k + 1
    candidate_trace = prefix + candidate suffix

    Score candidate_trace

    old_trace_score = average risk across current steps
    new_trace_score = average risk across candidate steps

    improvement = old_trace_score - new_trace_score

    If improvement < 0:
        reject candidate trace
        stop with reason "risk_worsened"

    If 0 <= improvement < improvement_threshold:
        accept candidate trace
        stop with reason "improvement_below_threshold"

    If improvement >= improvement_threshold:
        accept candidate trace
        continue to next iteration

If max_iters is reached:
    stop with reason "max_iters"

Write final_trace, final_answer, final scores, final risks, and iteration logs

-Evaluation:
For each record in final output:
    original_answer = model_answer
    final_answer = final_answer or repaired_answer or model_answer
    gold_answer = gold_answer

    Compare original_answer to gold_answer
    Compare final_answer to gold_answer

    Categorize answer outcome:
        wrong_to_correct
        wrong_to_wrong
        correct_to_correct
        correct_to_wrong

    Read original risk summary from iteration 0
    Read final risk summary from final_risks

    risk_delta = original_avg_risk - final_avg_risk

    Count repair-loop behavior:
        stop_reason
        number of repair attempts
        number of accepted repairs

Aggregate all records into metrics.json
Write per-example details to evaluation_records.jsonl
